# CodeMind AI 启动脚本使用说明

## 🚀 快速启动

### 方式 1：自动找端口启动（推荐）

```bash
cd backend
npm run auto
```

这个脚本会：
1. 🔍 自动寻找可用端口（从 5000 开始尝试）
2. 📝 自动更新 `.env` 文件的端口配置
3. 🔗 自动更新前端 API 地址
4. 🚀 启动后端服务器
5. 💬 提供交互式控制台

### 方式 2：普通启动

```bash
cd backend
npm run dev
```

使用 `.env` 文件中配置的端口（默认 3002）

### 方式 3：使用批处理文件（Windows）

```bash
cd backend
start.bat
```

## 📝 交互式控制台命令

使用 `npm run auto` 启动后，可以使用以下命令：

- `r` - 重启服务器
- `q` - 退出程序
- `h` - 显示帮助信息
- `Ctrl+C` - 快速退出

## 🔍 端口检测逻辑

脚本会按以下顺序尝试端口：

```
5000-5005
6000-6005
7000-7005
8000-8005
```

如果所有端口都被占用，会使用 5000 端口。

## 📁 文件说明

- `start.js` - 自动找端口启动脚本
- `start.bat` - Windows 批处理启动文件
- `.env` - 环境配置文件（包含端口设置）
- `package.json` - 项目配置文件

## 🔧 手动配置端口

如果需要指定端口，可以直接编辑 `.env` 文件：

```env
PORT=3002  # 修改这里
NODE_ENV=development
```

## 🌐 前后端端口

- **后端端口**：由 `.env` 文件或 `start.js` 自动设置
- **前端端口**：固定为 3000（在 Vite 配置中）

## ⚠️ 注意事项

1. **端口占用**：如果启动失败，检查端口是否被占用
2. **权限问题**：某些端口可能需要管理员权限
3. **防火墙**：确保防火墙允许所选端口的连接
4. **前端配置**：使用 `auto` 命令会自动更新前端 API 地址

## 🧪 测试连接

启动后，访问以下地址测试：

```bash
# 健康检查
http://localhost:<PORT>/health

# API 信息
http://localhost:<PORT>/api

# 前端界面
http://localhost:3000
```

## 💡 故障排除

### 端口都被占用怎么办？

1. 使用 `netstat -ano | findstr :<端口>` 查看占用端口的进程
2. 使用任务管理器结束占用进程
3. 或者使用 `npm run auto` 自动寻找可用端口

### 前端无法连接后端？

1. 检查后端是否正常启动
2. 确认前端 API 地址正确（应该是 `http://localhost:<PORT>/api`）
3. 检查浏览器控制台是否有 CORS 错误

### 修改端口后前端不更新？

使用 `npm run auto` 启动会自动更新前端 API 地址，或者手动修改 `frontend/src/pages/MainPage.tsx` 中的 API 地址。
