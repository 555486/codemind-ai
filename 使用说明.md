# 🚀 CodeMind AI - 使用说明

## ⚡ 快速启动 (推荐两种方式)

### 方式 1: 使用简化启动脚本 (最简单)

**步骤:**
1. 双击运行 `run.bat` 文件
2. 等待 10 秒
3. 浏览器自动打开 http://localhost:3001

**特点:**
- ✅ 自动清理旧进程
- ✅ 自动启动前后端
- ✅ 自动打开浏览器
- ✅ 无需手动操作

**位置:**
```
e:\trae\存项目\第二个代码优化项目\code-mind-ai\run.bat
```

---

### 方式 2: 手动启动 (适合开发)

**终端 1 - 启动后端:**
```bash
cd e:\trae\存项目\第二个代码优化项目\code-mind-ai\backend
npm run dev
```
看到 `🚀 CodeMind AI Backend Server Started!` 表示成功

**终端 2 - 启动前端:**
```bash
cd e:\trae\存项目\第二个代码优化项目\code-mind-ai\frontend
npm run dev
```
看到 `Local: http://localhost:3001/` 表示成功

**访问:** http://localhost:3001

## 🔧 常见问题解决

### 问题 1: 显示"无法连接到后端服务器"

**原因:** 后端没有启动或 CORS 配置错误

**解决方法:**
1. 检查后端是否启动成功
2. 查看后端日志是否有错误
3. 确认 `.env` 文件中的 `CORS_ORIGIN=http://localhost:3001`
4. 重启后端服务器

**验证:**
```bash
# 在浏览器访问
http://localhost:5000/health

# 应该看到:
{"status":"healthy","service":"CodeMind AI Backend",...}
```

---

### 问题 2: 端口被占用

**错误信息:** `Port 3000 is in use` 或 `Port 5000 is in use`

**解决方法 1 - 使用 run.bat (自动清理):**
```bash
双击运行 run.bat
```

**解决方法 2 - 手动清理:**
```bash
# Windows 命令提示符 (管理员)
taskkill /F /IM node.exe
```

**解决方法 3 - 修改端口:**
```bash
# 编辑 backend/.env
PORT=5001  # 改为其他端口

# 编辑 frontend 中的 API 地址
# 将所有 http://localhost:5000 改为 http://localhost:5001
```

---

### 问题 3: AI 功能用不了

**检查步骤:**

1. **检查 API Key 配置**
   ```bash
   # 查看 backend/.env 文件
   DASHSCOPE_API_KEY=sk-xxxxx  # 应该是 sk-开头
   ```

2. **检查后端日志**
   ```
   看到 "✅ DashScope API 密钥已配置" 表示配置正确
   看到 "📡 调用 DashScope API" 表示正在调用
   看到 "✅ DashScope API 调用成功" 表示成功
   ```

3. **测试 API**
   ```bash
   # 浏览器访问
   http://localhost:5000/health
   ```

4. **查看前端控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签
   - 查看 Network 标签中的 API 请求

---

### 问题 4: 点击"分析代码"跳转到空白页面

**原因:** Monaco 编辑器功能冲突

**解决方法:**
1. 刷新页面 (Ctrl+F5 强制刷新)
2. 清除浏览器缓存
3. 确保使用最新版本的代码

---

### 问题 5: 使用统计不更新

**原因:** 统计功能需要正确调用 API

**解决方法:**
1. 确保 AI 功能正常工作
2. 每次成功调用 AI 功能都会更新统计
3. 刷新页面查看最新统计

---

## 📖 使用教程

### 1. 代码分析

**步骤:**
1. 选择编程语言 (如 C++, JavaScript 等)
2. 在编辑器中输入代码
3. 点击左侧"分析代码"按钮
4. 等待 4-6 秒
5. 查看右侧"分析结果"面板

**示例代码:**
```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello World" << endl;
    return 0;
}
```

---

### 2. 代码解释

**步骤:**
1. 输入或粘贴代码
2. 点击左侧"解释代码"按钮
3. 等待 AI 生成解释
4. 查看右侧"AI 解释"面板

---

### 3. AI 教学

**步骤:**
1. 在"AI 教学"输入框输入主题
2. 点击发送按钮
3. 等待 AI 生成教学内容
4. 示例主题: "闭包", "Promise", "递归"

---

### 4. 复制代码

**步骤:**
1. 点击右上角"复制"按钮
2. 代码自动复制到剪贴板
3. 可以粘贴到其他地方

---

**文件:** `frontend/src/styles/global.css`

```css
:root {
  --neon-blue: #00d4ff;      /* 主色调 */
  --neon-green: #00ff88;     /* 强调色 */
  --neon-purple: #9d4edd;    /* 辅助色 */
```

---

### 修改 AI 响应速度

**文件:** `backend/src/services/ai.service.ts`

```typescript
parameters: {
  max_tokens: 800,      // 减少输出长度 (更快)
  temperature: 0.3,     // 降低创造性 (更稳定)
  top_p: 0.8
}
```

## 🔍 调试技巧

### 查看后端日志

**位置:** 后端启动的终端窗口

**关键信息:**
- `✅ DashScope API 密钥已配置` - API Key 正常
- `🔍 使用真实 AI 分析...` - 正在处理请求
- `📡 调用 DashScope API` - 正在调用 AI
- `✅ DashScope API 调用成功` - AI 调用成功
- `POST /api/analyze 200` - API 响应成功

---

### 查看前端日志

**步骤:**
1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 点击"分析代码"按钮
4. 查看输出信息

**关键信息:**
- `分析代码请求：xxx, 长度 xx 字符` - 请求已发送
- 如果有错误，会显示具体错误信息

---

### 网络请求调试

**步骤:**
1. 按 F12 打开开发者工具
2. 切换到 Network 标签
3. 点击"分析代码"按钮
4. 查看请求详情

**正常请求:**
- URL: `http://localhost:5000/api/analyze`
- Method: `POST`
- Status: `200`
- Response: `{success: true, results: [...]}`

## 📊 性能优化

### 加快 AI 响应

1. **减少输出长度**
   - 修改 `max_tokens: 800` (默认 1000)

2. **降低创造性**
   - 修改 `temperature: 0.3` (默认 0.5)

3. **使用更快的模型**
   - 如果阿里云有其他模型可选

---

### 提高前端性能

1. **生产构建**
   ```bash
   cd frontend
   npm run build
   ```

2. **启用缓存**
   - 配置浏览器缓存策略

3. **压缩资源**
   - 使用 gzip 或 brotli
